# Pattern #95 — unguarded-deserialize allowlist (task #265).
#
# This file lists deserialize sites that the gate
# (scripts/ci/detect_unguarded_deserialize.py) flagged but that have been
# audited and accepted as legitimate exceptions, OR that are grandfathered
# debt being fixed by tasks #264 / #266.
#
# Two entry forms are accepted:
#
#   1. ``severity|file|line|T_simple`` — line-locked allowlist key. Pull
#      these directly from the ``allowlist_key`` field of
#      ``docs/qa/unguarded-deserialize-report.json``. Moving the line
#      forces a fresh entry (intentional — re-review on movement).
#
#   2. Bare ``relative/path.cs`` — suppress every site in that file. Use
#      sparingly, only for files where the cross-FFI risk has been
#      fully addressed by an alternative mechanism (e.g. the file
#      validates each field manually before use, or the file lives at
#      a trust boundary that pre-validates upstream).
#
# Lines starting with ``#`` and blank lines are ignored.
#
# Run the gate locally with:
#   python scripts/ci/detect_unguarded_deserialize.py --quiet
# Self-test it with:
#   python scripts/ci/detect_unguarded_deserialize.py --self-test
# CI workflow: .github/workflows/unguarded-deserialize.yml
#
# Pairs with src/SDK/Validation/JsonGuard.cs + IValidatable.cs (the API
# the gate enforces). Tracking task: #265.
