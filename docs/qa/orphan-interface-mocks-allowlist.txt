# Pattern #125: Orphan Interface Mocks Allowlist
# Interfaces with 3+ production implementations but 0 test doubles
# are flagged as HIGH violations unless explicitly allowlisted below.
#
# Format: <InterfaceName> | <Reason>
# Single-line comments with #
#
# Allowlist Criteria:
# - Interface is documented as not requiring mocks (DSL, config-only)
# - Interface design intentionally avoids testing via seams (observation-only)
# - Test doubles would add complexity without coverage benefit
# - Bridge pattern actively discourages mocking via sealed impl
