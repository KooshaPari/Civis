=== CRITICAL FINDING: FastMCP HTTP Transport Model ===

FastMCP v3.x with --http flag uses Server-Sent Events (SSE), NOT direct JSON-RPC POST.

FastMCP HTTP Architecture:
  1. Claude Code (MCP Client) opens SSE connection to http://127.0.0.1:8765/
  2. Server streams messages back via SSE (multipart/event-stream)
  3. Client sends JSON-RPC 2.0 via SSE events, not direct HTTP POST
  4. Protocol is still JSON-RPC 2.0, but transport is SSE-based

Current Endpoints on http://127.0.0.1:8765:
  - GET  /health       (simple health check - returns JSON)
  - POST /hmr          (hot-module-reload trigger)
  - SSE  /             (MCP client connection - not direct JSON-RPC)

Why direct JSON-RPC POST to / returns 404:
  - FastMCP root endpoint expects SSE upgrade, not raw JSON-RPC
  - This is intentional; FastMCP clients handle SSE framing
