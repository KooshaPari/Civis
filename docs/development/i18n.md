# Internationalization (i18n) Guide

DINOForge supports multi-language user interfaces through a JSON-based localization system. All text strings are externalized from code and organized by language locale.

## Supported Locales

The following locales are currently supported:

| Locale Code | Language | File | Status |
|-------------|----------|------|--------|
| `en-US` | English (United States) | `assets/i18n/en-US.json` | Base language |
| `de-DE` | German (Germany) | `assets/i18n/de-DE.json` | Complete |
| `fr-FR` | French (France) | `assets/i18n/fr-FR.json` | Complete |
| `es-ES` | Spanish (Spain) | `assets/i18n/es-ES.json` | Complete |
| `zh-CN` | Simplified Chinese | `assets/i18n/zh-CN.json` | Complete |
| `ja-JP` | Japanese | `assets/i18n/ja-JP.json` | Complete |
| `pt-BR` | Portuguese (Brazil) | `assets/i18n/pt-BR.json` | Complete |
| `ru-RU` | Russian | `assets/i18n/ru-RU.json` | Complete |

## File Structure

Each locale file is a JSON object with string keys and translated values. Keys use dot notation to organize related strings:

```json
{
  "menu.title": "Mods",
  "menu.search.placeholder": "Search packs...",
  "menu.button.enable": "Enable",
  "debug.fps": "FPS: {0}"
}
```

### Key Naming Conventions

Keys are organized by feature and context:

- **`menu.*`** - Main mod menu UI strings
  - `menu.title` - Menu title
  - `menu.search.*` - Search interface
  - `menu.filter.*` - Filter options and labels
  - `menu.sort.*` - Sort options and labels
  - `menu.button.*` - Button labels and actions
  - `menu.detail.*` - Pack detail panel information
  - `menu.profile.*` - Profile management
  - `menu.update.*` - Update notifications
  - `menu.telemetry.*` - Telemetry settings
  - `menu.settings.*` - User settings
  - `menu.error.*` - Error messages
  - `menu.info.*` - Informational messages
  - `menu.notify.*` - User notifications

- **`debug.*`** - Debug information and stats
  - `debug.title` - Debug panel title
  - `debug.entities.count` - Entity count
  - `debug.systems.count` - System count
  - `debug.fps` - Frames per second
  - `debug.memory.*` - Memory statistics
  - `debug.cache.*` - Cache statistics
  - `debug.archetypes` - Archetype count
  - `debug.gc.*` - Garbage collection stats

### Placeholder Syntax

Translatable strings may contain placeholders for dynamic values. Use `{0}`, `{1}`, etc. for positional arguments:

```json
{
  "menu.update.version": "Version {0} available",
  "menu.profile.confirm_delete": "Delete profile '{0}'?"
}
```

**Important:** Always preserve placeholder syntax exactly as it appears. Do not translate or remove placeholders.

## How Translations Are Loaded

The localization system loads locale files dynamically at runtime:

1. **Locale Detection** - The system determines the user's locale (from OS settings or user preference)
2. **File Loading** - The corresponding JSON file is loaded from `assets/i18n/`
3. **Fallback** - If a locale is not available, the system falls back to `en-US.json`
4. **String Lookup** - When a UI element needs a string, it queries the loaded locale by key

### Example (Pseudo-code)

```csharp
// Load the user's locale
var userLocale = GetUserLocale(); // e.g., "de-DE"
var localeData = LoadJsonFile($"assets/i18n/{userLocale}.json");

// Get a translated string
var text = localeData["menu.title"]; // "Mods" → "Mods" (German)
```

## Adding a New Locale

To add support for a new language:

1. **Create a new JSON file** in `assets/i18n/` with the locale code:
   ```
   assets/i18n/it-IT.json  (Italian)
   assets/i18n/ko-KR.json  (Korean)
   ```

2. **Copy the structure** from `en-US.json` and translate every key:
   ```json
   {
     "menu.title": "[Italian translation]",
     "menu.search.placeholder": "[Italian translation]",
     ...all 72 keys...
   }
   ```

3. **Validate the file**:
   - Ensure all 72 keys are present (copy from en-US.json if needed)
   - Ensure valid JSON syntax (no trailing commas, properly quoted strings)
   - Preserve all `{0}` placeholders exactly as-is

4. **Test locally**:
   - Set the game's language to the new locale
   - Verify strings display correctly in the UI
   - Check for text overflow or layout issues

5. **Update documentation**:
   - Add the locale to the "Supported Locales" table in this file
   - Update any locale-specific code (L10n.cs) if needed

6. **Commit**:
   ```bash
   git add assets/i18n/[locale].json docs/development/i18n.md
   git commit -m "feat(i18n): add [locale] localization"
   ```

## Translation Conventions

When translating strings, follow these conventions for consistency:

### Gaming Terminology

- **Mods** - Use the gaming-standard term for your language (usually stays as "Mods" or translates to the equivalent gaming term)
- **Packs** - Mod packs are referred to as "packs" (German: "Packs", French: "packs", Spanish: "paquetes", Chinese: "包", etc.)
- **Enable/Disable** - Standard UI toggles (German: "Aktivieren"/"Deaktivieren", French: "Activer"/"Désactiver", etc.)

### Technical Terms

Some technical terms should be left in English or use the gaming-community standard:

- **FPS** - Frames Per Second (typically unchanged)
- **Heap** - Memory heap (may translate or leave as "Heap" depending on language norms)
- **Archetype** - ECS archetype (typically unchanged)
- **GC** - Garbage Collection (typically unchanged)

### Button and Label Conventions

- **Action buttons** - Use clear verbs (Enable, Disable, Save, Cancel, Delete)
- **Descriptive labels** - Use nouns or noun phrases (Profile, Language, Version)
- **Placeholders** - Use `...` to indicate optional input (Save As...)

### Formatting

- **Consistency** - Use consistent capitalization and punctuation across all strings
- **Punctuation** - Match the original English: if it ends with a colon, the translation should too
- **Quotes** - Use proper quotation marks for your language (`'`, `"`, « », etc.)

## Validating Locale Files

All locale files must:

1. **Have valid JSON syntax** - Use a JSON validator to check
2. **Contain exactly 72 keys** - Match the number of keys in en-US.json
3. **Preserve placeholders** - All `{0}` markers must remain unchanged
4. **Be UTF-8 encoded** - Use UTF-8 (with or without BOM) for special characters

### Validation Script

To validate all locales:

```bash
# Check all locale files for completeness
python3 -c "
import json

with open('assets/i18n/en-US.json') as f:
    en_us = json.load(f)
en_us_keys = set(en_us.keys())

for locale in ['de-DE', 'fr-FR', 'es-ES', 'zh-CN', 'ja-JP', 'pt-BR', 'ru-RU']:
    with open(f'assets/i18n/{locale}.json') as f:
        data = json.load(f)
    locale_keys = set(data.keys())
    
    missing = en_us_keys - locale_keys
    if missing:
        print(f'{locale}: MISSING {missing}')
    else:
        print(f'{locale}: OK ({len(locale_keys)} keys)')
"
```

## Implementation Details

The `L10n.cs` class (or equivalent) handles locale loading and string lookup:

```csharp
public class L10n
{
    private static Dictionary<string, string> _currentLocale;
    
    public static void LoadLocale(string localeCode)
    {
        var path = $"assets/i18n/{localeCode}.json";
        var json = File.ReadAllText(path);
        _currentLocale = JsonConvert.DeserializeObject<Dictionary<string, string>>(json);
    }
    
    public static string Get(string key, params object[] args)
    {
        if (_currentLocale.TryGetValue(key, out var template))
        {
            return string.Format(template, args);
        }
        return key; // Fallback to key name if translation missing
    }
}
```

Usage in UI:

```csharp
// Get a simple string
string title = L10n.Get("menu.title");

// Get a string with placeholders
string message = L10n.Get("menu.update.version", "1.2.0");
// Result: "Version 1.2.0 available"
```

## Best Practices

1. **Keep en-US as source of truth** - Always update en-US.json first, then propagate to other locales
2. **Test in-game** - Launch the game with each locale enabled to verify display
3. **Watch for text overflow** - Some languages (German, Russian) produce longer text; test UI layouts
4. **Community translations** - Consider crowdsourcing translations from the modding community
5. **Version control** - Track locale file changes in git; use meaningful commit messages

## Troubleshooting

### Missing Strings in Game

If a string doesn't appear translated in the game UI:
1. Check that the key exists in the locale file
2. Verify the key name matches exactly (case-sensitive)
3. Check that the locale file is valid JSON
4. Restart the game to reload locale files

### Text Overflow in UI

If translated text is too long for the UI container:
1. Use shorter translations if possible
2. Adjust the UI layout to accommodate longer text
3. Test with the longest-text locales first (German, Russian)

### Placeholder Not Replaced

If `{0}` appears in the game UI instead of the actual value:
1. Verify the placeholder syntax is preserved in the translation
2. Check that the code calling the string is using `string.Format()` or equivalent
3. Ensure the translation has the same number of placeholders as the English version
