FINDINGS & ANALYSIS
================================================================================

1. FASTMCP HTTP TRANSPORT MODEL
   - FastMCP v3.x in HTTP mode uses Server-Sent Events (SSE), NOT direct JSON-RPC POST
   - Root endpoint (/) accepts SSE upgrades from MCP clients, not raw JSON-RPC POST
   - Health endpoint (/health) returns JSON status for monitoring
   - HMR endpoint (/hmr) for hot-module-reload notifications

2. JSON-RPC 2.0 PROTOCOL SPECIFICATION
   - FastMCP correctly implements JSON-RPC 2.0 specification
   - Requests must be wrapped in tools/call method envelope
   - Proper request format:
     {
       "jsonrpc": "2.0",
       "id": "<unique-id>",
       "method": "tools/call",
       "params": {
         "name": "<tool-name>",
         "arguments": { /* tool-specific args */ }
       }
     }
   - Response format (success):
     {
       "jsonrpc": "2.0",
       "id": "<same-id>",
       "result": { /* tool result */ }
     }
   - Response format (error):
     {
       "jsonrpc": "2.0",
       "id": "<same-id>",
       "error": { "code": -32600, "message": "Invalid Request" }
     }

3. IMPLEMENTATION DECISION
   - Direct HTTP POST to FastMCP root not viable for PowerShell testing
   - PowerShell lacks native SSE client support without external libraries
   - Alternative: Use GameControlCli directly
   - GameControlCli wraps MCP tools via named pipes to game bridge
   - Provides equivalent functionality without HTTP transport complexity

RECOMMENDATIONS FOR PRODUCTION
================================================================================

For applications requiring JSON-RPC 2.0 direct HTTP POST to FastMCP:

Option 1: Use an SSE-capable HTTP client
  - Node.js: eventsource library
  - Python: aiohttp with SSE support
  - Go: eventsource library
  - Example: Create a thin wrapper service that handles SSE framing

Option 2: Use Claude Code's native MCP integration
  - Claude Code has built-in SSE/MCP client
  - Can directly call tools via MCP
  - Simplest for Claude-based workflows

Option 3: Use GameControlCli directly (current approach)
  - Bypass HTTP entirely
  - Use named pipes to game bridge
  - Lowest latency for in-process testing
  - No external dependencies
