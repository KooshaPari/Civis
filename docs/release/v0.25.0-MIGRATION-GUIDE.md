# DINOForge v0.25.0 Migration Guide (for NuGet consumers)

Target packages: `DINOForge.SDK`, `DINOForge.Bridge.Client`, `DINOForge.Bridge.Protocol`  
Version focus: upgrade to `0.25.0`

**No breaking changes:** The release keeps binary compatibility in common pipelines, but source usage may need updates in a few API patterns.

## What changed and what you should do

### 1) Pattern #226: `JsonRpcMessage` moved to properties

`JsonRpcMessage` and related JSON-RPC types now use public properties instead of public fields.

- Affected types:
  - `JsonRpcMessage`
  - `JsonRpcRequest`
  - `JsonRpcResponse`
  - `JsonRpcError`
- Binary compatibility is preserved through `[JsonPropertyName]` on properties.
- If you reflect over fields (for example `GetType().GetField("id")`), update to property access (`GetProperty("Id")`) or avoid reflection.

### 2) Pattern #367: `GameClient` now implements `IDisposable`

`GameClient` now implements `IDisposable` explicitly in `DINOForge.Bridge.Client`.

- Update callers to dispose `GameClient` explicitly.
- Use scope disposal (recommended) for short-lived clients: `using` statements or `using` declarations.
- Long-lived client instances should be disposed on shutdown or cancellation path.

### 3) Pattern #99 family: default comparer behavior changed to `StringComparer.Ordinal`

`Registry<T>` and `PackDependencyResolver` now default to `StringComparer.Ordinal` for lookups.

- Consumers that relied on case-insensitive matching should now pass `StringComparer.OrdinalIgnoreCase` when creating their own lookup dictionaries.
- This brings deterministic behavior and aligns with current compatibility guidance for package APIs.

### 4) Pattern #124: sealed SDK NuGet classes

Several SDK surface classes are now `sealed`.

- If your code previously subclassed SDK types, update to composition/composition-based adapters instead.
- Replace inheritance extension points with wrapper types that delegate to the sealed types.

### 5) Required package/version update

- Update your package references to `0.25.0`:
  - `DINOForge.SDK`
  - `DINOForge.Bridge.Client`
  - `DINOForge.Bridge.Protocol`
- In your `.csproj`, bump package versions to `0.25.0`.

## Recommended actions

1. Update all three package versions to `0.25.0`.
2. Add/restore disposal for every `GameClient` usage (`using` where appropriate).
3. Scan reflection code for `JsonRpcMessage`-related field access and migrate to properties.
4. Audit any `Registry<T>`/`PackDependencyResolver` consumers for case-insensitive assumptions.
5. Remove inheritance from now-sealed SDK types and refactor to composition.
6. Run your normal build and tests after package restore.

## Quick migration checklist

- If you pass IDs from user input into registry/dependency lookups, use `StringComparer.OrdinalIgnoreCase` only when needed.
- If you wrote subclasses of SDK classes, convert to wrapper types now (before touching business logic).
- If you create custom JSON-RPC serialization helpers, verify compatibility through public property names, not field names.
